# Proxy Beg

Proxy Beg - минимальное расширение для Chrome, которое помогает быстро включать, выключать и проверять прокси-серверы с авторизацией.

## Возможности

- Автоматическая загрузка актуального списка прокси.
- Ручное обновление прокси кнопкой `Refresh`.
- Добавление собственных HTTP или SOCKS5 прокси в формате `ip:port@username:password`.
- Бессрочное хранение прокси, добавленных пользователем, в `chrome.storage.local`.
- Выбор активного прокси из общего списка.
- Включение и выключение прокси прямо из popup-окна расширения.
- Проверка доступности и пинга всех прокси кнопкой `Check all`.
- Отображение статуса проверки: online, offline, задержка и время последней проверки.
- Определение страны прокси по IP.
- Автоматическая передача логина и пароля прокси через `webRequest.onAuthRequired`.

## Как установить

1. Скачайте или склонируйте проект на компьютер.
2. Откройте Chrome и перейдите на страницу `chrome://extensions`.
3. Включите режим разработчика в правом верхнем углу.
4. Нажмите `Load unpacked` или `Загрузить распакованное расширение`.
5. Выберите папку проекта `ProxyBeg`.
6. После установки нажмите на иконку расширения в панели Chrome.

## Как пользоваться

1. Откройте popup-окно расширения.
2. Дождитесь автоматической загрузки прокси или нажмите `Refresh` для ручного обновления.
3. При необходимости добавьте свой прокси в формате `127.0.0.1:1080@user:pass`.
4. Выберите нужный сервер из списка.
5. Включите прокси переключателем в верхней части окна.
6. Нажмите `Check all`, чтобы проверить доступность серверов и увидеть задержку.

## Формат прокси

Используйте формат:

```text
ip:port@username:password
```

Пример:

```text
127.0.0.1:1080@user:pass
```

Если провайдер не указывает протокол, сначала попробуйте HTTP. Chrome поддерживает SOCKS5 через `chrome.proxy`, но SOCKS5 с авторизацией в расширениях может работать менее стабильно, чем HTTP-прокси с авторизацией.

## Примечание

Когда расширение включено, выбранный прокси применяется глобально для обычного профиля Chrome.
